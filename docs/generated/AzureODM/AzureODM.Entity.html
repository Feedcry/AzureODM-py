save(force_replace=False, force_merge=False, force_save=False, ignore_conflict=False)[source]¶

save the entity to server

possible actions:

  • force_save to override _is_changed

  • insert (avoiding unknowleged replacing/merging) insert_entity()

    • if _is_new is True

    • if ignore_conflict is True,

      azure.WindowsAzureConflictError, will be ignored

  • insert or replace (update) insert_or_replace_entity()

    • if _is_new is True
    • and force_replace is True

  • insert or merge insert_or_merge_entity()

    • if _is_new is True
    • and force_merge is True

  • merge merge_entity()

    • if _is_new is False
    • and if _is_partial is True
    • or force_merge is True

  • replace (update) update_entity()

    • if _is_new is False
    • and if _is_partial is False
    • or force_replace is True

we need to check the validation and check changes

by calling _pre_save()

Parameters:
  • force_replace (bool) – default = False
  • force_merge (bool) – default = False
  • force_save (bool) – default = False
_after_save(saved_entity_dict)[source]¶

called after save function, if successful

some functions will return a full entity_dict, we will call _populate_with_dict()

  • insert_entity

some functions will only return a dict with only ‘etag’ key, we will manually call _copy_into_saved() and change status variables

  • update_entity
  • merge_entity
  • insert_or_replace_entity
  • insert_or_merge_entity
  • change _is_new to False (via _populate_with_dict())
  • change _is_changed to False (via _populate_with_dict())
  • change _saved_etag (via _populate_with_dict())
Parameters:

saved_entity_dict (dict) – could also be AzureEntity
Raises:
  • TypeError – if saved_entity_dict is other,
  • KeyError – if etag is not found in dict
Returns:

True if everything is fine
